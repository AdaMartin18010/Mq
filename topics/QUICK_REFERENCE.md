## 📊 核心参数对比

### 消息模型

- **Kafka**: 发布-订阅 + 队列（Consumer Group）
- **MQTT**: 发布-订阅（Topic）
- **NATS Core**: 发布-订阅 + 请求-响应
- **NATS JetStream**: 发布-订阅 + 流

### 协议特性

- **Kafka**: 自定义二进制协议
- **MQTT**: OASIS标准（3.1.1/5.0），2字节固定头
- **NATS**: 文本协议，简单易调试

### 集群模式

- **Kafka**: Leader-Follower + ZooKeeper
- **MQTT**: 主从/集群（共享存储）
- **NATS**: 全网状自愈（Gossip协议）

## 🎯 选型决策树（简化版）

```
需要持久化？
├─ 是 → 吞吐量 > 100万TPS?
│   ├─ 是 → Kafka
│   └─ 否 → NATS JetStream
└─ 否 → 延迟要求 < 1ms?
    ├─ 是 → NATS Core
    └─ 否 → IoT设备?
        ├─ 是 → MQTT
        └─ 否 → NATS Core
```

## 📈 监控关键指标

### Kafka黄金指标

- UnderReplicatedPartitions（欠副本分区）
- ConsumerLag（消费延迟）
- ISRShrinkRate（ISR收缩速率）

### MQTT黄金指标

- ConnectedClients（连接数）
- MessageDropRate（消息丢弃率）
- KeepAliveTimeout（心跳超时率）

### NATS黄金指标

- Routes（集群路由数）
- SlowConsumers（慢消费者）
- MemoryUsage（内存占用）
